# 画面仕様書（ブログ機能）

本書は `app/blog/` 配下の画面、および入力値の検証を担う `actions/blog-schema.ts` を対象とした画面仕様書です。

## 対象範囲

| 画面ID | 画面名 | パス | 種別 |
| --- | --- | --- | --- |
| SCR-BLOG-01 | ブログ新規登録画面 | `/blog/create` | Server Component |
| SCR-BLOG-02 | ブログ詳細画面 | `/blog/[id]` | Server Component |
| SCR-BLOG-03 | ブログ編集画面 | `/blog/[id]/edit` | Server Component |

ブログ一覧画面は `/top` に存在するため本仕様書の対象外。各画面とも未ログイン時は `/` へリダイレクトされる。

---

## SCR-BLOG-01 ブログ新規登録画面

### 概要
ログイン中のユーザーがブログを新規投稿する画面。

### URL
- `/blog/create`

### アクセス制御
| 条件 | 挙動 |
| --- | --- |
| 未ログイン | `/` にリダイレクト |
| ログイン中 | 画面表示 |

### 画面構成
| エリア | 部品 | 内容 / 動作 |
| --- | --- | --- |
| ヘッダーリンク | テキストリンク「← 一覧に戻る」 | クリックで `/top` へ遷移 |
| カード見出し | `CardTitle` | 「新規登録」 |
| カード説明 | `CardDescription` | 「ブログを投稿します」 |
| 入力フォーム | `BlogForm` | タイトル / 本文 / 送信ボタン |

### 入力項目
| 項目 | 型 | 必須 | 制約 | エラーメッセージ |
| --- | --- | --- | --- | --- |
| タイトル | テキスト | ○ | 1〜255文字 | 「タイトルを入力してください」 |
| 本文 | テキストエリア（6行） | ○ | 1〜10000文字 | 「本文を入力してください」 |

### ボタン
| 名称 | 表示文言 | 押下時 |
| --- | --- | --- |
| 送信ボタン（通常時） | 登録 | サーバーアクション `createBlog` を呼び出す |
| 送信ボタン（送信中） | 送信中… | 押下不可（`disabled`） |

### バリデーション動作
- 初期表示時は検証しない
- フォーカスアウト時（`onBlur`）で初回検証
- 入力中（`onInput`）で再検証
- サーバー側でも `blogFormSchema` で再検証

### 正常時遷移
- 登録成功 → `/top` に再検証付きリダイレクト（`revalidatePath('/top')` → `redirect('/top')`）

### エラー表示
| 種別 | 表示位置 | 内容 |
| --- | --- | --- |
| 項目エラー | 各入力欄直下 | スキーマ違反メッセージ |
| フォーム全体エラー | フォーム最上部 | 「ログインが必要です」「登録に失敗しました。時間をおいて再度お試しください」 |

## SCR-BLOG-02 ブログ詳細画面

### 概要
個別のブログ記事を表示する画面。本文は Markdown としてレンダリング（`react-markdown` + `remark-gfm`）。

### URL
- `/blog/[id]`（`id` は正の整数）

### アクセス制御
| 条件 | 挙動 |
| --- | --- |
| 未ログイン | `/` にリダイレクト |
| `id` が正の整数でない | `notFound()`（404） |
| 該当ブログが存在しない | `notFound()`（404） |
| ログイン中かつ該当ブログ存在 | 画面表示 |

### 画面構成
| エリア | 部品 | 表示条件 / 内容 |
| --- | --- | --- |
| ヘッダーリンク | テキストリンク「← 一覧に戻る」 | クリックで `/top` へ遷移 |
| タイトル | `<h1>` | `blog.title` |
| 操作ボタン群 | 編集ボタン / 削除ボタン | **ログインユーザー == 投稿者** の場合のみ表示 |
| メタ情報 | `<p>` | 「{投稿者名 または 不明} ・ {作成日（YYYY/M/D）}」 |
| 本文 | `<article>` | Markdown レンダリング |

### 操作ボタン
| 名称 | アイコン | 押下時 |
| --- | --- | --- |
| 編集ボタン | `Pencil` | `/blog/{id}/edit?from=detail` へ遷移 |
| 削除ボタン | `Trash2` | 確認ダイアログ（`AlertDialog`）を開く |

### 削除確認ダイアログ
| エリア | 内容 |
| --- | --- |
| タイトル | 「削除しますか?」 |
| 説明 | 「『{タイトル}』を削除します。この操作は取り消せません。」 |
| キャンセルボタン | ダイアログを閉じる |
| 削除ボタン（通常時） | サーバーアクション `deleteBlog` を呼び出す |
| 削除ボタン（送信中） | 「削除中…」、両ボタン押下不可 |

### 正常時遷移
- 削除成功 → `/top` に再検証付きリダイレクト

## SCR-BLOG-03 ブログ編集画面

### 概要
自分が投稿したブログを更新する画面。

### URL
- `/blog/[id]/edit`
- クエリ: `from=list` または `from=detail`（戻り先制御）

### アクセス制御
| 条件 | 挙動 |
| --- | --- |
| 未ログイン | `/` にリダイレクト |
| `id` が正の整数でない | `notFound()`（404） |
| 該当ブログが存在しない | `notFound()`（404） |
| ログインユーザー != 投稿者 | `/blog/{id}` にリダイレクト |
| 投稿者本人 | 画面表示 |

### 画面構成
| エリア | 部品 | 内容 / 動作 |
| --- | --- | --- |
| ヘッダーリンク | テキストリンク | `from=list` → 「← 一覧に戻る」 `/top`<br>それ以外 → 「← 詳細に戻る」 `/blog/{id}` |
| カード見出し | `CardTitle` | 「編集」 |
| カード説明 | `CardDescription` | 「ブログを更新します」 |
| 入力フォーム | `BlogForm` | 現在値で初期化、送信ボタン文言は「更新」 |

### 入力項目
SCR-BLOG-01 と同じ（`blogFormSchema`）。

### ボタン
| 名称 | 表示文言 | 押下時 |
| --- | --- | --- |
| 送信ボタン（通常時） | 更新 | サーバーアクション `updateBlog(id, ...)` を呼び出す |
| 送信ボタン（送信中） | 更新中… | 押下不可 |

### 正常時遷移
- 更新成功 → `/top` および `/blog/{id}` を再検証 → `/blog/{id}` にリダイレクト

### エラー表示
| 種別 | 表示位置 | 内容 |
| --- | --- | --- |
| 項目エラー | 各入力欄直下 | スキーマ違反メッセージ |
| フォーム全体エラー | フォーム最上部 | 「ログインが必要です」「更新権限がないか、対象が見つかりませんでした」「更新に失敗しました。時間をおいて再度お試しください」 |

## 共通仕様

### レイアウト
- 全画面 `<main>` を中央寄せ、上下に余白（`px-4 py-10`）
- 詳細画面: `max-w-2xl`、それ以外: `max-w-lg`（フォーム部 `max-w-md`）

### スタイル
- `components/ui/*`（Button / Input / Textarea / Label / Card）を使用
- 削除ダイアログは `@base-ui/react/alert-dialog`、アイコンは `lucide-react`
- 詳細画面の本文は `prose` クラスでマークダウンを装飾、ダークモード対応（`dark:prose-invert`）

### バリデーションスキーマ（`actions/blog-schema.ts`）
```ts
blogFormSchema = z.object({
  title: z.string().min(1, 'タイトルを入力してください').max(255),
  body:  z.string().min(1, '本文を入力してください').max(10000),
});
```
